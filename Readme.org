#+author: lollinus
#+options: toc:2

* Overview

=emacs-bazel= extends [[https://github.com/bazelbuild/emacs-bazel-mode][bazel.el]] (the official Bazel Emacs package) with IDE
features inspired by [[https://github.com/bazelbuild/vscode-bazel][vscode-bazel]]:

- *compile_commands.json generation* — uses a Bazel aspect to extract
  compiler flags from =cc_*= targets and assembles =compile_commands.json=
  for clangd / eglot.
- *Per-project configuration* — persistent package lists and Bazel config
  selection (debug/release flags, toolchain configs).
- *GDB debugging* — build =cc_test= / =cc_binary= targets with debug flags,
  locate the binary, and launch GDB with proper environment variables
  (=RUNFILES_DIR=, =TEST_SRCDIR=, =TEST_TMPDIR=).

The goal is to provide the core Bazel C++ development workflow entirely
within Emacs, without requiring VSCode or any proprietary extensions.

* Features

** Compile Commands Generation

Uses a [[https://bazel.build/extending/aspects][Bazel aspect]] to traverse the build graph and extract compile
information from =CcInfo= providers and the C++ toolchain — no actual
compilation required.

Supports two modes:
- *Per-directory* — writes a =compile_commands.json= in each source
  package directory.  clangd discovers it by walking upward.
- *Single file* — writes one =compile_commands.json= at a configured
  location (for use with =--compile-commands-dir=).

** Package / Target Selection

Projects can be large (100k+ targets).  =emacs-bazel= lets you
configure which packages to index:

#+begin_src json
{
  "packages": ["//my/activity/...", "//platform/core/..."],
  "configurations": {
    "clang-debug": ["--config=clang19", "--cxxopt=-g", "--cxxopt=-O0", "--strip=never"]
  },
  "activeConfiguration": "clang-debug",
  "mode": "per-directory"
}
#+end_src

Interactive commands allow adding/removing packages and switching
configurations with persistence.

** Build and Test Commands

- =emacs-bazel-build= — prompts for a target (with =bazel.el= completion)
  and configuration, runs =bazel build= in =compilation-mode=
- =emacs-bazel-test= — same, filtered to test targets
- Configuration selection via =emacs-bazel--select-configuration= reads
  from the project's =config.json=
- With =C-u= prefix, =emacs-bazel-refresh= also lets you pick a
  configuration for the aspect build

** Debug with GDB

- Builds the target with debug flags (=-c dbg= or custom config)
- Resolves the binary path via =bazel cquery --output=files=
- Sets up =RUNFILES_DIR=, =TEST_SRCDIR=, =TEST_TMPDIR= for =cc_test=
- Launches =gud-gdb= with the workspace root as source directory
- Supports =--gtest_filter= via test-at-point detection

* Requirements

- Emacs ≥ 29.1
- [[https://github.com/bazelbuild/emacs-bazel-mode][bazel.el]] (provides workspace detection, target completion, =bazel-command=)
- Bazel ≥ 6.0 (for =cc_common= APIs used by the aspect)
- clangd (for consuming =compile_commands.json= via eglot)
- GDB (for debug features)

* Installation

Clone this repository into your Emacs =load-path=:

#+begin_src shell
git clone git@github.com:lollinus/emacs-bazel.git ~/projects/emacs-bazel
#+end_src

Then in your Emacs config:

#+begin_src elisp
(leaf emacs-bazel
  :load-path "~/projects/emacs-bazel"
  :after bazel
  :commands (emacs-bazel-refresh
             emacs-bazel-build
             emacs-bazel-test
             emacs-bazel-debug-target
             emacs-bazel-test-at-point)
  :bind (("C-c b r" . emacs-bazel-refresh)
         ("C-c b b" . emacs-bazel-build)
         ("C-c b t" . emacs-bazel-test)
         ("C-c b d" . emacs-bazel-debug-target)
         ("C-c b T" . emacs-bazel-test-at-point)))
#+end_src

* Per-Project Setup

On first use in a project, =emacs-bazel-refresh= will:
1. Create =.bazel-compdb/= directory in the project root
2. Deploy =cpp-info-aspect.bzl= there
3. Create a default =config.json=

Add =/.bazel-compdb/= to your =.gitignore=.

* Known Issues and Notes

** Aspect Path Requires BUILD File

The =.bazel-compdb/= directory must contain an empty =BUILD= file.
Bazel resolves aspect paths as labels (=//.bazel-compdb:cpp-info-aspect.bzl=)
and labels require the directory to be a valid Bazel package.  The setup
command creates this automatically.

** =.bazelrc= Config Conflicts

If your =~/.bazelrc= or project =.bazelrc= sets a default =--config== value
(e.g., =--config=ipn_gcc9=) and you select a different configuration in
=emacs-bazel=, Bazel will emit warnings like:

#+begin_example
WARNING: option '--platforms' was expanded to from both
  option '--config=ipn_gcc9' (source ~/.bazelrc) and
  option '--config=ipn_clang' (source command line)
#+end_example

This is expected.  Command-line flags take precedence, so the build uses
your selected configuration correctly despite the warnings.

** Running Tests

#+begin_src shell
emacs -batch -l ert -l emacs-bazel.el -l emacs-bazel-test.el \
